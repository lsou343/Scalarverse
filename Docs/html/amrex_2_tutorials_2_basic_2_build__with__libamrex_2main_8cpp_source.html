1//
2// This example demonstrates:
3//
4// (1) By default, AMReX initializes MPI and uses MPI_COMM_WORLD as its communicator.
5// However, applications could choose to initialize MPI themselves and pass in an
6// existing communicator.
7//
8// (2) By default, AMReX treats command line arguments as inputs parameters. The expected
9// format of argv is
10// executable inputs_file parm=value
11// Here, `excutable` is the filename of the executable, `inputs_file` is the file containing
12// runtime parameters used to build AMReX ParmParse database, and `parm=value` is an input
13// parameter that will override its value in `inputs_file`. Both `inputs_file` and
14// `parm=value` are optional. At most one `inputs_file` is allowed. Howeer, there can be
15// multiple `parm=value`s.
16//
17// The parsing of the command line arguments is performed in amrex::Initialize. Applications
18// can choose to skip command line parsing. Applications can also provide a function that
19// adds parameters to AMReX ParmParse database.
20//
